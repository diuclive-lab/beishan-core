### 1. 关键词统一归档

**新建 `plugins/intent_keywords.go`**，作为所有意图判断词表的唯一来源。

词表分七层：
- `cmdConfirmWords / cmdForceSaveWords / cmdMergeWords` — 系统指令精确匹配
- `cmdBatchPrefix` — 批量确认前缀
- `cmdRememberTriggers / cmdReviewTriggers / cmdListReviews / cmdConfirmAll / cmdSkipAll` — 功能触发词
- `ctxCurrentSession` — 指向"刚刚发生的事"（不检索）
- `ctxCrossSession` — 指向历史 session（episodic 检索）
- `ctxSemantic` — 指向知识库结论（semantic 检索）
- `ctxVagueRef` — 模糊指代（需 LLM 改写再检索）
- `ctxCode` — 代码相关（代码检索优先）

**删除以下原始定义**（共 13 个变量）：

### 2. OnMessage 职责拆分

`think_plugin.OnMessage` 从 178 行缩减到 **26 行纯分发器**。

引入两个新函数：
- `isSystemCommand(text, sessionID) bool` — 判断是否系统指令（8 个检查合一）
- `handleSystemCommand(text, sessionID)` — 执行所有状态机操作，不做检索、不调 LLM

分叉原则：`isSystemCommand` 为 true → 进状态机；否则 → `handleChat`。两条路径完全独立。

内部引入 `chatReply` 辅助函数，消除 `handleSystemCommand` 里 11 处重复的 `json.Marshal` 模板。

### 3. Session 读取走正规路径

`loadRecentSessionMessages` 原来：
```go
tools.ValidateAndExecute("read_file", rawMsg(map[string]interface{}{
    "path": filepath.Join(tools.MemoryDir, "sessions", sessionID+".json"),
}))
```

改为：
```go
tools.SessionGet(sessionID)
```

`SessionGet` 持有 `sessionMu.RLock()`，并发安全。同时删除了只在这一行使用的 `filepath` import。

### 4. Session 摘要自动生成（接通断路）

在 `cmd/beishan/main.go` 的 assistant 回复落盘后，加入后台 goroutine：

```go
go func(sid string) {
    time.Sleep(500 * time.Millisecond) // 等待 session_add 消息落盘
    if sum := tools.GenerateSessionSummary(sid); sum != nil {
        tools.SaveSessionSummary(sum)
    }
}(sessionID)
```

500ms 延迟确保 `k.Send(session_add)` 经过 kernel 路由到 memory_plugin 后已写盘。
生成的 `{sessionID}.summary.json` 供 `SessionSearchStructured Phase 1` 使用。

### 5. review_handler 文件系统调用规范化

`deleteReviewFile`：删除无意义的 `ValidateAndExecute("patch", 空 old_string/new_string)` 调用（该调用不删文件，静默失败）。保留 `os.Remove`，加注释说明无 delete_file 工具。

## 设计原则沉淀

本次修复确立了几条可复用原则，已写入知识库：

1. **单一词表原则**：关键词只在 `intent_keywords.go` 定义，其他文件只引用。修改词义只改一处。
2. **OnMessage 分发原则**：OnMessage 是纯路由，不含任何业务逻辑。状态机和自然语言严格分叉。
3. **Session 读取原则**：必须通过 `tools.SessionGet`，不直接操作文件路径。
4. **摘要生成原则**：每次对话结束后异步生成 session 摘要，确保跨会话检索可用。
