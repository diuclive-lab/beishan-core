### 发现的问题

| 问题类型 | 涉及文件 | 严重程度 |
|----------|----------|:--------:|
| 成功路径返回空 Message（Payload=nil） | browser_plugin, image_gen_plugin, tts_plugin, write_plugin (4 cases) | **高** — 调用方收不到结果 |
| ValidateAndExecute 后未检查 result.Success | browser_plugin, image_gen_plugin, tts_plugin, write_plugin, search_plugin (web_fetch), skill_factory (skill_evaluate) | **中** — 失败时返回误导性 `.result` 类型 |
| 用 `result.Error != ""` 代替 `!result.Success` | notify_plugin | **低** — 语义不等价 |
| 失败时返回 `.result` 类型而非 `.error` | todo_plugin, skill_factory (skill_evaluate), search_plugin (fallback), notify_plugin (已修复) | **中** — 调用方无法区分成功/失败 |

### 修复汇总

**browser_plugin.go / image_gen_plugin.go / tts_plugin.go：**
- 之前：`result := tools.ValidateAndExecute(...)` → `return kernel.Message{}, nil`
- 之后：`!result.Success` 检查 + 编码 Output 为 JSON Payload + 返回 `msg.Type+".result"`

**write_plugin.go：**
- write_file/read_file/search_files/patch 四个 case 从空 Message 改为输出 payload
- 全部 case 添加 `!result.Success` 检查

**notify_plugin.go：**
- `result.Error != ""` → `!result.Success`（两者语义不同）

**search_plugin.go：**
- web_fetch/web_extract/web_render 添加 `!result.Success` → `.error` 类型
- web_search 降级路径添加 `.error` 类型判断

**todo_plugin.go + skill_factory_plugin.go：**
- 失败时返回 `msg.Type+".error"` 而非 `.result`

## 新发现的 AI 错误模式（已更新至 §9）

1. **Plugin 返回空 Message** — `return kernel.Message{}, nil` 使调用方收不到结果。检测：搜索 `return kernel.Message{}, nil` + 非 default 分支。
2. **ValidateAndExecute 后未检查 result.Success** — 语法正确但语义不完备。检测：搜索 ValidateAndExecute 后面是否有 .Success 引用。
3. **用 `result.Error != ""` 代替 `!result.Success`** — ErrorResult 同时设置两者，但自定义 handler 可能只设 Success=false。
4. **workflow YAML 缺少 on_error/timeout** — v2.5 强制项，漏了可能使工作流挂死。检测：grep 对比步骤数量与 on_error/timeout 数量。
5. **workflow YAML 缺少 output_target** — 引擎默认 chat，但不声明违反规范。检测：grep -L output_target。
6. **非标准步骤格式 YAML 混入 workflows/** — 引擎会执行空 plugin 步骤，静默失败。检测：检查 step 的 plugin 字段是否存在。
