### 架构：切换流程

```
启动时
  └─ discovery.ScanWithModel() → 发现本地引擎
  └─ 选择 fallback 候选（优先 OpenAI-compatible）
  └─ 启动 monitorFailover goroutine（每 30s）

监控循环
  └─ checkAPIReachable() + checkEngineReachable()
  └─ state.Decide(apiOK, localOK)  // 带滞后防抖
  └─ 切换到 local:
       ├─ llm.SetProvider("local")
       ├─ k.Router.SetStrategy(LocalRouteStrategy)
       └─ os.Setenv("LLM_BASE_URL", engine.Endpoint)
  └─ 切回 api（滞后 2 次成功）:
       ├─ llm.SetProvider("")    // 恢复默认
       ├─ k.Router.SetStrategy(nil)
       └─ os.Unsetenv("LLM_BASE_URL")
```

### 关键设计决策

- **LocalRouteStrategy 持有 Router 引用** — 复用 buildPluginList() + parseDecision()，硬化层校验完全一致
- **线程安全 SetProvider** — RWMutex 保护 overrideProvider，不影响活跃请求
- **OpenAI-compatible 优先** — selectFallbackEngine 优先选 openai/llamacpp 类型（支持 /v1/chat/completions）
- **LLM_BASE_URL/LLM_MODEL 环境变量覆盖** — 切换时设/清环境变量，ChatCompletionWithUsage 自动适配

### 后续改进：代码审查驱动的三处优化

根据代码审查建议，验证并实现了三点改进：

#### 1. Prompt 模板适配（LocalRouteStrategy）

本地模型（Qwen、llama.cpp 等）对 JSON 格式的生成可靠性低于 DeepSeek。在 `callLocal` 中增加 system message：

```go
systemMsg := `You are a routing engine. Your ONLY output must be a single valid JSON object.
No markdown, no code fences, no explanations, no greetings, no additional text.
Output format: {"recipient":"plugin_name","msg_type":"message_type","payload":{},"reason":"why","confidence":0.0}
If you are unsure, set confidence to 0.0.`
```

仅对 LocalRouteStrategy 生效，不影响默认 DeepSeek 路由。

#### 2. 负索引防护（extractFieldFromValue）

`internal/workflow/engine.go` 中解析点号路径的数组索引时，`fmt.Sscanf` 的返回值被忽略。改为显式校验：

```go
if n, err := fmt.Sscanf(part[braceStart+1:len(part)-1], "%d", &arrIdx); err != nil || n != 1 || arrIdx < 0 {
    return nil, false
}
```

负索引或非整数字符现在被显式拒绝，而非静默返回整个数组。

#### 3. 切换事件写入 Observatory Trace

新增 `observatory.SetDefaultRecorder` / `RecordTrace` 全局接口，main.go 初始化持久化 Recorder：

```go
observatory.SetDefaultRecorder(observatory.NewPersistentRecorder(
    filepath.Join("eval/run/traces/traces_20260524.jsonl")))
```

`monitorFailover` 在切换和恢复时各写入一条 Trace：

- `failover_switch` — API 不可用，切换到本地模型（含引擎名和模型名）
- `failover_recovery` — API 恢复，切回 DeepSeek

存入 `eval/run/traces/traces_YYYYMMDD.jsonl`，与决策 Trace 共享同一持久化机制。

### 核心认知

1. **CI 不应该因为配置问题全盘失败** — smoke 测试 offline 模式让配置缺失和代码问题解耦。API key 没配不表示代码坏了，offline 仍能验证编译和硬化完整。这个模式值得推广到其他 CI 步骤。

2. **事先设计的接口减少了集成风险** — RouteStrategy 接口和 StrategyState 状态机是方案阶段定义的，今天做集成时零改动。成本花在方案设计阶段，收益在实现阶段。

3. **线程安全 Provider 切换比环境变量更可靠** — 最初想用 `os.Setenv("LLM_PROVIDER", "local")`，但环境变量读取没有并发控制。改用 RWMutex 保护的 overrideProvider，不影响正在进行的请求。
