### 四、工作流引擎 Bug 修复

**问题：** `resolveJSONValue` 递归无深度限制，处理 `code_tree` 大 JSON 输出时栈溢出。

**根因：** engine.go:620-622 对 JSON 字符串做解包递归，嵌套的大 JSON 导致无限递归。

**修复：** 新增 `resolveJSONValueDepth(raw, depth int)`，depth > 10 时截断返回原始数据。

**影响：** 直接影响 `code_deep_analyze` Step 1（code_tree list_files）和 `quick_project_scan`（parallel + memory_plugin）。

### 五、Agent 系统修复（来自 P2 吸收缺口分析）

使用 v2.5 工作流对已吸收的 P2 Agent 委派做后验缺口分析，发现 7 个遗漏：

| # | 类别 | 严重度 | 修复 |
|---|------|--------|------|
| 1 | 并发安全 | critical | currentSpawnDepth 全局 int → context-based depth |
| 2 | 错误分类 | high | SubagentResult.Error string → ErrorCode 枚举 |
| 3 | 截断通知 | high | tool 结果截断时标注 [truncated: X/Y chars] |
| 4 | 输出校验 | medium | 检测空输出/放弃响应 |
| 5 | LLM 重试 | medium | 网络抖动自动重试 1 次 |
| 6 | 异步写盘 | medium | conversation persistence goroutine 异步 |
| 7 | 空 prompt | low | RunSubagent 入口前置校验 |
