# 开发日志 2026-05-21

## content_review 工作流修复（6 个 bug）

### Bug 1: web_render 工具未注册

`tools.go` 的 `Init()` 缺少 `registerWebRenderTool()` 调用，导致 workflow 的 fetch 步骤找不到工具。

### Bug 2: Array→String 类型强转

`schema_registry.go` 期望 string 但收到 `[]interface{}` 时，取首元素或 JSON 序列化。空数组→空字符串。

### Bug 3: no_retrieval 模式

新增 `ModeNoRetrieval`（review_handler.go），workflow 步骤跳过检索直接调 LLM。think_plugin 的 mode dispatch 提升到自然语言触发器之前，避免页面内容 "Skip to main content" 误触发 isSkipAll。

### Bug 4: search_queries[0] 索引失效

`engine.go` 的 `extractJSONFieldValue` 在 map 取值后未立即应用数组索引。修复：`m[fieldName]` 后立即检查 `arrIdx`。

### Bug 5: 空数组处理

空 `search_queries: []` 导致 schema 校验失败。schema_registry.go 添加空数组→空字符串 coercion。

### Bug 6: buildPayload 空输入返回字符串

workflow 无 input 时，`ctx["input"]` 为 `""`，`buildPayload` 返回 `"{}"`（JSON string）而非 `{}`（JSON object）。修复：检查 `input != ""`。

## 核心认知

1. **workflow 无 input 时 buildPayload 必须返回 `{}`** — `ctx["input"]` 为 `""` 不等于不存在，必须显式检查空字符串。
2. **mode dispatch 必须在自然语言触发器之前** — 页面内容可能包含触发词（"skip"），但用户意图是 workflow 步骤。
3. **入库门禁用软修正不用硬拒绝** — 自动补 source_type/tags 比拒绝入库更实用。
