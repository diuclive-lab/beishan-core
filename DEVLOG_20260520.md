## 已知问题

### 问题 1：路由决策不稳定
- 同一问题多次测试，路由器可能分发到不同插件
- 例："hermes-go项目的核心架构" → 有时 think_plugin，有时 search_plugin
- 原因：DeepSeek 路由的非确定性输出
- 影响：用户体验不一致

### 问题 2：知识注入无引用验证
- 知识上下文注入 system prompt 后，无法验证 LLM 是否真正使用
- LLM 可能忽略知识上下文，用自己的训练数据回答
- 可能混合知识库内容和"幻觉"
- 需要：引用追踪或置信度评估机制

### 问题 3：知识检索阈值
- BOW 向量 + 类型权重已工作，但相似度阈值需要调优
- 当前阈值：0.15 (KnowledgeRetrieve) vs 0.25 (KnowledgeSemanticSearch)
- 可能漏掉相关但表述不同的知识条目

## 下一步

## 技术细节

### 硬化检索算法
```
1. 用户输入 → textToVector (BOW, 512维)
2. 遍历所有知识条目
3. 加载预计算的 embedding
4. 计算 cosineSimilarity
5. 乘以 typeWeight (Principle=1.3, ADR=1.2, Lesson=1.1, ...)
6. 排序返回 top N
```

### 知识类型权重
```go
var typeWeight = map[string]float64{
    "Principle":   1.3,  // 稳定原则，优先参考
    "ADR":         1.2,  // 架构决策，重要参考
    "Lesson":      1.1,  // 经验总结
    "AntiPattern": 1.1,  // 禁止行为
    "Hotspot":     1.0,  // 代码风险点
    "Telemetry":   0.9,  // 指标反馈
    "":            1.0,  // 未分类
}
```
